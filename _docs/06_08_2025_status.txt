#### Что было сделано:

   - При создании трейда записываются оба статуса: "Трейд создан в Steam" и актуальный статус

2. **Обновлен скрипт очистки базы данных**

3. **Полностью переработан прогресс-бар заказов**

     - Последний активный шаг показывает оранжевый вопросик
     - Отмененные шаги показывают красный крестик
   - Добавлена анимация стрелок для активного шага

#### Текущая логика работы:

1. TradeOfferObserver автоматически записывает все изменения статусов в trade_offer_status_history
2. При wasChanged('steam_trade_offer_id') дополнительно записывается "Трейд создан в Steam"
3. Используется проверка уникальности timestamp для избежания конфликтов первичного ключа
4. Order модель имеет accessor trade_status_history который возвращает историю для фронтенда

#### Важные файлы:
- /app/Observers/TradeOfferObserver.php - логика записи истории
- /app/Models/TradeOfferStatusHistory.php - модель истории
- /resources/js/components/profile/Orders.vue - компонент с прогресс-баром
- /resources/scss/pages/_profile.scss - стили прогресс-бара (строки 665-794)
- /database/migrations/2025_08_05_111733_create_trade_offer_status_history_table.php - миграция

#### Структура таблицы trade_offer_status_history:
- trade_offer_id (foreign key)
- status (string)
- created_at (timestamp)
- Составной первичный ключ: [trade_offer_id, created_at]

#### Известные особенности:
- При одновременной записи нескольких статусов используется addMicroseconds() для уникальности
- История сохраняется в "сыром" виде, маппинг статусов происходит только для отображения
- Прогресс-бар динамически адаптируется под количество шагов в истории
