## 🔄 СЛЕДУЮЩАЯ ЗАДАЧА - УПРАВЛЕНИЕ ДУБЛИРУЮЩИМИ ТРЕЙДАМИ

### 📋 Проблема:
Steam позволяет создавать множественные предложения с одним предметом:
- Один предмет → несколько активных trade offers
- Покупатели видят дубли
- Принятие любого трейда делает остальные недействительными

### 🎯 Решение - Серверный мониторинг с принудительной отменой:

#### Алгоритм обработки трейда:

1. **Инициация (Сервер)**:
   - Резервирует предмет на 5 минут
   - → WebSocket: `"create_trade"` (order_data)

2. **Создание (Расширение)**:
   - Создает предложение обмена в Steam
   - ← WebSocket: `"trade_created"` (tradeofferid) 

3. **Мониторинг (Сервер)**:
   - Каждые 30 сек проверяет статус через Steam API
   - Если статус "Accepted" → успех, помечаем выполненным
   - Если 5 минут прошло → принудительная отмена

4. **Отмена по таймауту (Сервер)**:
   - → WebSocket: `"cancel_trade"` (tradeofferid)

5. **Выполнение отмены (Расширение)**:
   - Пытается отменить трейд в Steam
   - ← WebSocket: `"trade_cancel_result"` (success: true/false)

6. **Финализация (Сервер)**:
   ```javascript
   if (cancelResult.success) {
       // Отмена прошла - снимаем резерв
       unreserveItem(itemId);
   } else {
       // Отмена не прошла - проверяем финальный статус
       finalStatus = checkSteamAPI(tradeofferid);
       if (finalStatus === "Accepted") {
           // Трейд принят в последний момент
           markOrderCompleted(orderId);
       }
   }
   ```

### 📋 WebSocket события для реализации:

#### Сервер → Расширение:
