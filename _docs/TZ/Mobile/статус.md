## Готово (в рамках мобильной переверстки)
- **2.1 Бургер-меню** — мобильный drawer в `MobileHeader`
- **2.2 Онлайн-счётчик** — реализован полностью: реальный счёт через middleware (Redis ZSET), Reverb broadcast раз в 10 сек на канал `online`, UI на десктопной и мобильной шапке, админ-страница `/admin/online-counter-settings` с режимами (real / real+fake / fake), random walk и суточным профилем
- **2.3 Мобильная версия**
  - Маркетплейс (главная, карточки, корзина, профиль) — переверстана
  - Кейсы:
    - Главная (категории, поиск, сетка)
    - CaseDetails (превью, footer с множителями, кнопки)
    - Upgrade (трио верх + табы списков «Ваши/Цели», стек предметов с закладками и подъёмом наверх)
    - CaseInventory (табы, баланс-блок, карточки, история апгрейдов со стеком ставок)
  - Универсальный `MobileBottomNav` со своим набором пунктов для cases: Маркет / Кейсы / Апгрейд / Соцсети / FAQ
  - Mobile-balance блок только на странице инвентаря кейсов
- **3.1 Мобильный фильтр маркетплейса** — шторка, кнопки Применить/Сбросить
- **5.1 Пагинация (числа + 25/50/100)** — общий компонент `resources/js/shared/components/Pagination.vue` (light/dark варианты). Применён на: маркетплейс, аукционы (заменили «Загрузить ещё»), покупки/продажи в профиле (с табами + counts с сервера), баланс (Операции/Бонусы/Удержание), история входов Premium, CaseInventory (Items + история апгрейдов), Inventory профиля (клиентская), Избранное (клиентская). Кейсы главная/категории — намеренно без пагинации
- **4.2 Свечение + hover-увеличение кейсов** — на hover карточка увеличивается (`scale(1.05)`) и подсвечивается тенью; цвет тени настраивается в админке (`CaseModelResource` → ColorPicker `glow_color`), пробрасывается через CSS-переменную `--case-glow-color` (по умолчанию `#ff8731`)
- **6.1 Детальная статистика активаций промокодов** — в карточке промокода: секция статистики (общая сумма пополнений/бонусов, кол-во активаций, новые/старые с первым/повторным промокодом) + RelationManager «Активации» с таблицей (пользователь со ссылкой на профиль, сумма пополнения, бонус, итог, дата). В карточке пользователя — RelationManager «Активированные промокоды» с теми же колонками и ссылкой на промокод
- **3.2 Апгрейд: привязка к предметам маркетплейса** — `SiteSetting upgrade_target_mode` (`virtual`|`market`). В режиме `market` пул целей формируется из активных листингов от онлайн-продавцов (фильтр как на маркетплейсе), группировка по `market_hash_name` с минимальной ценой. При выполнении апгрейда `firstOrCreate` `VirtualItem` по `market_hash_name`, дальше стандартный флоу. Переключатель — modal-action на странице `Кейсы → Апгрейды` в Filament, текущий источник отображается в лейбле кнопки
- **6.2 Генерация скрытых URL на кейсы** — таблица `case_secret_links` (token, label, expires_at, max_visits, visits_count, is_active), Filament-ресурс «Скрытые ссылки» в группе «Кейсы» (создание, копирование URL, edit/delete), публичный роут `GET /cases/secret/{token}` вне auth-группы — валидирует токен, инкрементит счётчик и рендерит `CaseController::index()` для гостя
- **6.3 Рекламные менеджеры + 6.4 Менеджеры партнёров + 6.9 Централизованный журнал действий** — установлены `bezhansalleh/filament-shield` 4.2 (RBAC + UI ролей) и `spatie/laravel-activitylog` 4.x. На `User` подключены `HasRoles` + `FilamentUser`. Сгенерированы 361 permission и 29 policies через `shield:generate`. Сидер `ManagerRolesSeeder` создаёт роли `ad_manager` (AdBanner CRUD, Analytics view, Promocode view/create/update, Client read-only) и `partner_manager` (Promocode view + страница «Мои партнёры»). Pivot `user_partner` — привязка партнёров к менеджеру; `clients.withdraw_blocked` (boolean) для toggle-action «Остановить вывод» в `ClientResource`. Row-scoping в `PromocodeResource::getEloquentQuery()` для partner_manager. Кастомный Dashboard скрывает все виджеты для не-super_admin (кроме AccountWidget). На Pages добавлены trait `HasPageShield` (Analytics, OnlineCounterSettings) и `canAccess()` (MyPartners). Activity-log: trait `LogsActivity` на `Client`/`Promocode`/`AdBanner` + read-only Filament-ресурс `ActivityLogResource` с фильтрами по action/user/date/тип объекта. Страница «Мои партнёры» — нативные Filament-секции с фильтром по промокоду, статистикой переходов/активаций/пополнений/бонусов
- **6.6 Топ-пользователи (подозрительная активность)** — Filament-страница `SuspiciousActivity` в группе «Отчёты» (super_admin only): период (час/24ч/неделя/произвольный), порог подсветки ₽, лимит строк. Запрос на лету: брутто-сумма выпавших скинов из `case_opens`+`case_inventory_items` (без `is_free`) + сумма `target_price` из выигрышных `upgrades`, GROUP BY client. Колонки: пользователь со ссылкой на профиль, кейсы/апгрейды/итого ₽, операций. Подсветка строки danger при превышении порога. Без SiteSetting и фоновых джоб
- **6.10 Комментарий админа в профиле клиента** — добавлено `clients.admin_comment` (text nullable), Textarea «Комментарий админа» в форме редактирования клиента, изменения логируются в activity_log
- **6.5 Запрет на вывод / предметы / баланс** — три раздельные блокировки на клиенте через `*_blocked_until` (datetime nullable) + `*_block_reason_admin/_user`. Кнопки «Заблокировать: Вывод/Предметы/Баланс» в шапке EditClient (модалка с DateTimePicker, toggle «Бессрочно», причина для админа и для клиента; при разблокировке показываются текущие значения readonly). Каскад: `balance` блокирует и `withdraw` и `purchases`. Гарды в WithdrawService, CaseController.purchaseCase, UpgradeService.execute, DepositController.createPaymentForm, OrderController (cartBuy/quickBuy/quickSell). На фронте Balance.vue: модалки вывода и пополнения подтягивают свежий статус через `/api/profile/me` при клике и показывают lock-экран с причиной из админки. MarketplaceController.getOnlineSellerIds исключает клиентов с активной блокировкой продаж — их листинги не появляются ни в маркете/категориях/featured, ни в пуле целей апгрейда (UpgradeService). Все изменения логируются в activity_log
- **6.7 Лимиты на вывод + заявки на одобрение** — таблица `balance_withdraw_requests` (client_id, amount, snapshot 24ч/1ч, limit_exceeded, status, processed_by/at). SiteSetting: `minimum_withdraw_amount` + 3 лимита (24ч/all, 24ч/user, 1ч/all; 0 = без ограничения). Endpoint `POST /api/profile/withdraw-balance` — проверки блокировок и лимитов, сразу замораживает сумму на балансе, создаёт запись Transaction(pending). При превышении лимита — флаг `limit_exceeded` и текст из ТЗ. Filament-страница «Заявки на вывод (баланс)» в группе «Финансы» (нав-бейдж кол-ва pending) с действиями Одобрить (Transaction→completed) и Отклонить (возврат на баланс + 48ч `balance_blocked_until` с авто-причиной + Transaction(refund)). Модалка `Balance.vue#balance-withdraw` — три состояния (заблокировано / результат / форма с минимумом и валидацией). Гарды также в `AuctionService::placeBid/createAuction`, `CaseInventoryController::sell`, `DepositController::activatePromocode`, `SubscriptionController::purchase`
- **4.3 Модалка пополнения при недостаточном балансе** — переиспользуемый `resources/js/shared/components/DepositModal.vue` (поле суммы, быстрые кнопки 100/500/1000/5000, промокод с debounce-валидацией, выбор платёжки). Подключена на странице кейса: при `(main + bonus) < price` открывается модалка вместо toast «Недостаточно средств», с предзаполненной нужной суммой. После успешного депозита локальный баланс обновляется и пользователь сразу может открыть кейс. Стили под тёмную тему через класс `theme-cases-modal`
