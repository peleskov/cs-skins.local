## Готово (в рамках мобильной переверстки)
- **2.1 Бургер-меню** — мобильный drawer в `MobileHeader`
- **2.2 Онлайн-счётчик** — реализован полностью: реальный счёт через middleware (Redis ZSET), Reverb broadcast раз в 10 сек на канал `online`, UI на десктопной и мобильной шапке, админ-страница `/admin/online-counter-settings` с режимами (real / real+fake / fake), random walk и суточным профилем
- **2.3 Мобильная версия**
  - Маркетплейс (главная, карточки, корзина, профиль) — переверстана
  - Кейсы:
    - Главная (категории, поиск, сетка)
    - CaseDetails (превью, footer с множителями, кнопки)
    - Upgrade (трио верх + табы списков «Ваши/Цели», стек предметов с закладками и подъёмом наверх)
    - CaseInventory (табы, баланс-блок, карточки, история апгрейдов со стеком ставок)
  - Универсальный `MobileBottomNav` со своим набором пунктов для cases: Маркет / Кейсы / Апгрейд / Соцсети / FAQ
  - Mobile-balance блок только на странице инвентаря кейсов
- **3.1 Мобильный фильтр маркетплейса** — шторка, кнопки Применить/Сбросить
- **5.1 Пагинация (числа + 25/50/100)** — общий компонент `resources/js/shared/components/Pagination.vue` (light/dark варианты). Применён на: маркетплейс, аукционы (заменили «Загрузить ещё»), покупки/продажи в профиле (с табами + counts с сервера), баланс (Операции/Бонусы/Удержание), история входов Premium, CaseInventory (Items + история апгрейдов), Inventory профиля (клиентская), Избранное (клиентская). Кейсы главная/категории — намеренно без пагинации
- **4.2 Свечение + hover-увеличение кейсов** — на hover карточка увеличивается (`scale(1.05)`) и подсвечивается тенью; цвет тени настраивается в админке (`CaseModelResource` → ColorPicker `glow_color`), пробрасывается через CSS-переменную `--case-glow-color` (по умолчанию `#ff8731`)
- **4.3 Модалка пополнения при недостаточном балансе** — переиспользуемый `resources/js/shared/components/DepositModal.vue` (поле суммы, быстрые кнопки 100/500/1000/5000, промокод с debounce-валидацией, выбор платёжки). Подключена на странице кейса: при `(main + bonus) < price` открывается модалка вместо toast «Недостаточно средств», с предзаполненной нужной суммой. После успешного депозита локальный баланс обновляется и пользователь сразу может открыть кейс. Стили под тёмную тему через класс `theme-cases-modal`
