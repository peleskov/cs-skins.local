# Техническое задание: CS2 Marketplace

## Обзор проекта

### Описание
Современная онлайн-платформа для торговли предметами CS2, объединяющая функционал маркетплейса и системы открытия кейсов. Разработка ведется в два этапа:
- **Этап 1**: Маркетплейс (P2P торговля и торговля через ботов)
- **Этап 2**: Система открытия кейсов

### Технологический стек
- **Backend**: Laravel
- **Frontend**: HTML, Bootstrap 5, SCSS
- **База данных**: MySQL/PostgreSQL
- **Языки**: Русский, Английский
- **Платформы**: Десктопная и мобильная версии сайта

## Этап 1: MVP Маркетплейса

### 1. Основной функционал

#### 1.1 Типы торговли

##### Быстрая продажа боту
- Мгновенный выкуп по цене ниже рыночной
- Расчет цены на основе Steam маркета:
  ```
  Цена выкупа = Минимальная цена Steam × Коэффициент редкости
  
  Коэффициенты редкости:
  - Ширпотреб: 50%
  - Промышленное качество: 50%  
  - Армейское качество: 40%
  - Запрещенное: 35%
  - Засекреченное: 30%
  - Тайное: 20%
  - Контрабанда: 20%
  ```
- Валидация: предмет должен иметь >200 лотов на Steam маркете
- Невалидные предметы отображают плашку "Не востребован"

##### Продажа на маркете (P2P)
- Пользователь устанавливает цену
- Требует установки браузерного расширения
- Комиссия платформы: 5% с продавца
- Таймаут трейда: 5 минут на подтверждение

#### 1.2 Роли пользователей

##### Обычные пользователи
- Могут покупать/продавать предметы
- Должны привязать Steam Trade URL для торговли
- Могут просматривать сайт без авторизации

##### Боты (специальные пользователи)
- Обычные аккаунты с флагом `is_bot`
- Автоматически принимают трейды
- Автоматическое управление инвентарем
- Дневной лимит: 200-300 трейдов на бота
- На старте: 5 ботов

#### 1.3 Финансовая система

##### Пополнение
- Несколько платежных систем
- Только для оплаты покупок (без пополнения баланса в MVP)
- Комиссия зависит от провайдера

##### Вывод средств
- Способы: банковская карта, баланс сайта с кейсами
- Минимум: $0.5
- Комиссия:
  - < 3334₽: 3.5% + 50₽
  - ≥ 3334₽: 5%
- Требуется верификация
- Требуется платежный пароль
- Время обработки: зависит от банка

### 2. Пользовательский интерфейс

#### 2.1 Публичные страницы
- **Маркетплейс**
  - Каталог предметов с фильтрами
  - Функция поиска
  - Корзина
  - Страница товара с историей цен
  
- **Фильтры**
  - Цена (поля ввода)
  - Тип (пистолеты, ножи, перчатки)
  - Качество (износ)
  - Редкость

#### 2.2 Личный кабинет
- **Торговля** - активные лоты пользователя
- **Инвентарь** - предметы пользователя
- **Мои аукционы** - управление аукционами
- **История покупок** - все транзакции
- **Настройки** - Trade URL, пароли
- **Пополнить баланс** - способы пополнения
- **Вывод** - вывод средств

#### 2.3 Дополнительные страницы
- FAQ/Помощь
- Правила и пользовательское соглашение
- Онлайн чат

### 3. Техническая реализация

#### 3.1 Steam интеграция
- Авторизация через Steam (обязательна для торговли)
- Получение инвентаря пользователя
- Отправка/прием трейд-офферов
- Проверка статуса трейдов (через таймауты, т.к. Steam убрал API)
- Получение данных о предметах

#### 3.2 База данных

##### Основные таблицы
```sql
- users (включая is_bot флаг)
- items (каталог всех предметов CS2)
- listings (активные предложения)
- trades (обмены)
- transactions (финансовые операции)
- bot_inventories (инвентари ботов)
- price_history (история цен)
```

##### Начальное наполнение
- Импорт всех предметов CS2 из wiki.cs.money
- Загрузка изображений предметов
- Категоризация по типу и редкости

#### 3.3 Безопасность
- Логирование всех операций
- Эскроу система (холд средств до подтверждения)
- Автоматическая отмена трейдов по таймауту
- Защита от мошенничества через проверку статусов

#### 3.4 Админ-панель
Настраиваемые параметры:
- Проценты выкупа по редкостям
- Комиссии платформы
- Минимальные/максимальные суммы
- Таймауты операций
- Количество предметов для валидации (по умолчанию 200)
- FAQ для пользователей

### 4. Браузерное расширение

#### 4.1 Функционал
- Авторизация в Steam
- Прослушивание команд с сервера
- Автоматическое создание трейдов
- Отображение статуса

#### 4.2 Поддержка
- Chrome/Edge (Chromium)
- Firefox
- Публикация в официальных магазинах

### 5. Ограничения и правила

#### 5.1 Торговля
- Минимальная цена предмета: 0.5₽
- Максимальное количество лотов: без ограничений
- Максимальная сумма продажи боту: 100,000₽ (настраивается)

#### 5.2 Пользователи
- Новые пользователи: нет доступа к аукционам
- Доступ к аукционам: после 1 покупки и 1 продажи
- Trade URL: не обязателен для просмотра

### 6. Внешние сервисы

#### 6.1 Источники данных
- **Каталог предметов**: https://wiki.cs.money/ru
- **API для интеграции**: https://market.csgo.com/ru/api
- **Цены**: Steam маркет + внутренние цены

#### 6.2 Платежные системы
- GMpays
- GameMoney
- Другие (уточняется)

8. **Система вывода** - 3-4 дня

#### 7.2 Приоритеты для MVP
1. Быстрая продажа боту
2. Базовый маркетплейс

#### 7.3 Можно отложить

- Мобильное приложение

### 8. Требования для запуска

#### 8.1 Steam боты
- 5 аккаунтов ($5 + телефон на каждый)
- 15 дней ожидания Steam Guard
- Настройка Trade URL

#### 8.2 Начальный инвентарь
- Бюджет: ~350,000₽
- Источник: csmarket
- Распределение: случайное между ботами

#### 8.3 Критерии готовности
- ✓ Вход через Steam
- ✓ Работающий маркетплейс с фильтрами
- ✓ Картинки скинов
- ✓ Аукционы (базовый функционал)
- ✓ Система пополнения/вывода
- ✓ Готовые боты с инвентарем
- ✓ Стабильная работа сайта
- ✓ Онлайн чат

## Этап 2: Система кейсов (будущее развитие)

### Основные функции
- Каталог кейсов с отображением содержимого
- Процентные шансы выпадения каждого предмета
- Лайв-лента выпадений в реальном времени
- Алгоритм расчета с "коэффициентом жадности"
- Возможность создания пользовательских кейсов
- История открытий

### Интеграция с маркетплейсом
- Выпавшие предметы можно продать/вывести/улучшить
- Использование цен с маркетплейса
- Переводы между балансами (если раздельные)

### Экономическая модель
- Гарантированный выигрыш в каждом кейсе
- Математическое ожидание <100% от стоимости кейса
- Регулируемые коэффициенты через админку

## Контактная информация и заметки

### Дизайн
- Готовый дизайн в Figma
- Адаптивная верстка обязательна

### Важные решения
1. MVP запускается с браузерным расширением (P2P не откладываем)
2. Боты реализованы как обычные пользователи с дополнительным функционалом
3. Все настройки через админ-панель
4. Двуязычный интерфейс с самого начала

### Риски
1. 15-дневная задержка Steam Guard для новых ботов
2. Лимиты трейдов (200-300 в день на бота)
3. Необходимость оборотного капитала
4. Steam API не предоставляет цены напрямую
5. Высокая конкуренция на рынке
